# 后续优化方向

## 短期

- 将 `ScreenshotAlertParser.extract_text_from_image()` 替换为真实 OCR 或多模态模型。
- 将 `badcases/` 从本地 JSON 升级到数据库或向量库。
- 增加 eval 自动运行脚本，自动调用接口并记录得分。
- 在前端展示 trace 列表和单次诊断详情。

## 中期

- 将统一 `ToolExecutor` 深度接入 LangGraph ToolNode，所有工具调用都走同一套治理。
- 为 replanner 增加证据完整性评分，避免过早结束。
- 对 bad case 做聚类，识别高频失败模式。
- 增加用户反馈按钮，把人工评价进入评测闭环。

## 长期

- 接入真实日志平台、监控平台、告警平台和工单系统。
- 引入多 Agent 协作，让日志分析、指标分析、知识库分析分别给出观点。
- 建设生产级权限体系，控制工具调用范围和敏感信息脱敏。
- 建设在线评测和灰度发布机制。

## 面试题

1. 如果继续做一个月，你会优先做什么？
2. 如何把本地 JSON bad case 升级为生产可用方案？
3. 多 Agent 会带来哪些收益和风险？
4. 接真实监控平台时要注意哪些安全问题？
5. 如何设计 Agent 的线上灰度发布？
