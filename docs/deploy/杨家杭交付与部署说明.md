# 杨家杭最终交付与部署说明

## 1. 最终交付内容

本交付包覆盖杨家杭在分工文档和实施步骤 v16 中的正式职责：

| 类别 | 文件/目录 | 说明 |
|---|---|---|
| Admin 任务编排 | `backend-java/knowledge-management/km-admin-service/` | 任务唯一创建入口、用户重试、Worker claim/heartbeat、结果幂等消费、REPROCESS 版本切换、PURGE 调度与审计 |
| Worker | `backend-java/knowledge-management/km-worker-service/` | PROCESS、REPROCESS、REEMBED、PURGE 消费；Redis 全局许可；心跳；延迟重试；FastAPI 契约调用；结果上报 |
| Gateway | `backend-java/gateway-service/` | `/api/v1/documents/**`、`/internal/km/**`、检索接口路由 |
| Search 占位 | `backend-java/knowledge-management/km-search-service/` | 接收配置变更广播，保证全量 Compose 联调完整 |
| AI 占位 | `backend-python/km-ai-service/` | FastAPI mock，提供 `/health`、解析、切片、向量化、向量删除契约 |
| RabbitMQ | `infra/rabbitmq/definitions.json`、`infra/rabbitmq/rabbitmq.conf` | `/km` vhost、主队列、结果队列、配置广播队列、DLX/DLQ、TTL 延迟重试队列 |
| Nacos | `infra/nacos/configs/` | Admin、Worker、Gateway 配置基线 |
| Nginx | `infra/nginx/nginx.conf` | 仅代理 `gateway-service:9000` |
| Docker Compose | `infra/docker/docker-compose.demo.yml` | MySQL、Redis、RabbitMQ、Nacos、MinIO、ChromaDB、AI、Admin、Worker、Search、Gateway、Nginx |
| SQL | `sql/*.sql` | 文档、任务、状态日志、幂等事件、版本、旧向量清理、PURGE 审计表 |
| 脚本 | `scripts/start-local.ps1`、`scripts/stop-local.ps1` | 全量启动、停止和可选清卷 |

## 2. 部署前准备

1. 安装 Docker Desktop，并确认 Docker 正在运行。
2. 打开 PowerShell，进入交付目录：

```powershell
cd "D:\桌面\软件工程\课程设计\项目"
```

3. 如 PowerShell 阻止脚本执行，临时放开当前窗口执行权限：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

## 3. 一键启动

在 `项目` 目录执行：

```powershell
.\scripts\start-local.ps1
```

脚本实际执行的是：

```powershell
docker compose -f infra/docker/docker-compose.demo.yml up -d --build
```

首次启动会拉取 MySQL、Redis、RabbitMQ、Nacos、MinIO、ChromaDB、Nginx 等镜像，并构建 Java/FastAPI 服务，时间会比较长。

## 5. 验证任务链路

1. 创建 PROCESS 任务：

```powershell
Invoke-RestMethod -Method Post `
  -Uri "http://localhost:9000/internal/km/tasks/process" `
  -ContentType "application/json" `
  -Body '{"docId":1,"userId":1}'
```

2. 查询任务状态：

```powershell
Invoke-RestMethod -Method Get `
  -Uri "http://localhost:9000/api/v1/documents/1/tasks"
```

3. 在 RabbitMQ 管理台检查以下队列均存在：

```text
km.doc.process
km.doc.reprocess
km.chunk.reembed
km.doc.purge
km.task.result
km.config.changed.worker
km.config.changed.search
km.doc.process.retry
km.doc.reprocess.retry
```

4. MySQL 中检查任务、状态日志和幂等事件：

```sql
select * from km_document_process_task order by id desc;
select * from km_document_status_log order by id desc;
select * from km_processed_event order by id desc;
```

## 6. 与其他成员模块的接入方式

上传模块在 MinIO 上传和 `km_document` 落库成功后调用：

```http
POST /internal/km/tasks/process
```

审核编辑切片后调用：

```http
POST /internal/km/tasks/reembed
```

审核拒绝后调用：

```http
POST /internal/km/tasks/review-reprocess
```

知识库策略变更后调用 `KnowledgeBaseReprocessService.createReprocessTasks(kbId, strategyVersion, operatorId)`，由 Admin 按 READY 文档游标分页创建 REPROCESS 任务。

普通用户失败重试调用：

```http
POST /api/v1/documents/{docId}/retry
```

查询文档任务历史调用：

```http
GET /api/v1/documents/{docId}/tasks
```

## 7. 停止与清理

保留数据停止：

```powershell
.\scripts\stop-local.ps1
```

停止并清空 MySQL、MinIO、ChromaDB 数据卷：

```powershell
.\scripts\stop-local.ps1 -PurgeData
```

## 8. 验收对应关系

- 上传自动入队：`TaskCommandService.createProcessTask()` + `afterCommit` 投递。
- 状态可查询：`GET /api/v1/documents/{docId}/tasks`。
- 失败重试：`POST /api/v1/documents/{docId}/retry`，最多 3 次。
- 审核编辑重向量化：`REEMBED` 队列与 Worker 分支。
- 策略变更批处理：`KnowledgeBaseReprocessService`。
- REPROCESS 安全切换：`km_document_version` + `target_version_no`。
- 单用户并发：Admin 创建任务时按用户统计 `QUEUED/RUNNING`。
- 全局解析并发：Worker 使用 Redis ZSET `km:processing:permits`。
- 延迟重试：`km.doc.process.retry`、`km.doc.reprocess.retry`。
- PURGE：`km_purge_audit` 独立审计，成功后物理删除业务记录。
- 配置刷新：`km.config.changed.worker` 与 `km.config.changed.search` 广播队列。
- 部署：`infra/docker/docker-compose.demo.yml` 全量启动。
